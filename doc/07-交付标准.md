# 三审制稿件工作台 - 交付标准（07）

---

## 1. PDF交付标准原文要求

请将项目打包或提供代码仓库链接，必须包含：

1. 前后端源代码  
   - 代码需结构清晰、命名规范，包含必要的注释。

2. 启动说明文档（README）  
   - 提供明确的依赖安装与启动方式。

3. 系统设计说明  
   - 简述前后端交互的 API 接口设计。  
   - 简述数据模型设计方案。  
   - 说明状态流转设计思路。
