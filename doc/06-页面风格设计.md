# 三审制稿件工作台 - 页面风格设计（06）

本文用于定义“三审制稿件工作台”前端视觉规范。风格参考 `https://www.sfccn.com/` 及其公开品牌描述，采用“清晰、明朗、大气”的财经媒体视觉方向，并结合本项目MVP页面（列表/编辑/详情审核）落地。

---

## 1. 设计目标

- 建立统一的页面视觉语言，保证列表、编辑、审核页风格一致。
- 对齐财经资讯产品气质：专业、权威、可信、信息密度适中。
- 在原生 `HTML + CSS + JavaScript` 条件下可直接实现，不依赖前端框架。

---

## 2. 视觉设计原则（必须遵守）

## 2.1 清晰（信息优先）
- 层级清晰：标题 > 关键信息 > 操作区 > 辅助信息。
- 内容块边界清晰：卡片化布局，适度留白。
- 状态识别清晰：状态标签颜色统一且语义固定。

## 2.2 明朗（阅读友好）
- 主背景使用高明度中性色，降低视觉噪声。
- 重要按钮强调对比，次要按钮弱化处理。
- 文本颜色分三层：主文本、次文本、说明文本。

## 2.3 大气（品牌一致）
- 页面网格规整，避免碎片化小组件堆叠。
- 关键视觉（头部、主按钮、状态条）使用品牌主色系。
- 图标与边框风格保持简洁，减少过度装饰。

---

## 3. 配色系统（仿照SFCCN风格）

| 色彩语义 | Token | 建议色值 | 用途 |
|----------|-------|----------|------|
| 主品牌金（南财金） | `--brand-gold` | `#C7A24F` | 顶部品牌条、Logo点缀、重点分割线 |
| 辅助绿（21世纪绿） | `--brand-green` | `#16A36A` | 成功态、通过态、正向提示 |
| 科技蓝 | `--brand-blue` | `#1F67D8` | 主按钮、链接、主交互 |
| 品牌红（基因红） | `--brand-red` | `#D14343` | 危险操作、退回、错误提示 |
| 主文本 | `--text-primary` | `#1F2A37` | 标题与正文 |
| 次文本 | `--text-secondary` | `#5B6675` | 描述与辅助信息 |

| 分割边框 | `--border` | `#E5EAF1` | 卡片边框、表格分割 |

---

## 4. 字体与排版规范

- 字体族：`"Microsoft YaHei", "PingFang SC", "Helvetica Neue", Arial, sans-serif`
- 页面宽度：内容区建议 `1200px`（桌面端），左右留白自适应
- 字号层级建议：
  - 一级标题：`28px / 700`
  - 二级标题：`22px / 600`
  - 区块标题：`18px / 600`
  - 正文：`14px / 400`
  - 说明文字：`12px / 400`
- 行高：
  - 标题 `1.3`
  - 正文 `1.6`

---

## 5. 组件样式规范

## 5.1 按钮

| 类型 | 背景 | 文本 | 边框 | 场景 |
|------|------|------|------|------|
| Primary | `--brand-blue` | `#fff` | 无 | 提交审核、审核通过、重新提交 |
| Success | `--brand-green` | `#fff` | 无 | 完成态确认、正向完成 |
| Danger | `--brand-red` | `#fff` | 无 | 审核退回 |
| Secondary | `#fff` | `--text-primary` | `--border` | 取消、返回、普通操作 |

交互要求：
- 悬浮：透明度降低或亮度提升 5%~8%
- 禁用：透明度 50%，`cursor: not-allowed`

## 5.2 状态标签（Badge）

| 状态 | 标签色 |
|------|--------|
| `DRAFT` | 灰蓝底 + 深灰字 |
| `PENDING_1ST` | 淡蓝底 + 蓝字 |
| `PENDING_2ND` | 淡青底 + 青字 |
| `PENDING_3RD` | 淡紫底 + 紫字 |
| `REJECTED` | 淡红底 + 红字 |
| `COMPLETED` | 淡绿底 + 绿字 |

## 5.3 卡片与表格
- 卡片圆角：`12px`
- 卡片阴影：`0 6px 20px rgba(31,42,55,0.06)`
- 表格头背景：`#F8FAFC`
- 行hover：`#F3F7FF`

## 6. 页面级设计要点（对应MVP三页）

## 6.1 稿件列表页
- 顶部：品牌栏 + 页面标题 + 快速筛选（按状态）
- 中部：列表卡片/表格展示（标题、状态、更新时间、操作）
- 操作强调：`新建稿件`与`查看详情`分主次按钮

## 6.2 稿件编辑页
- 左主区：标题输入 + 正文编辑区
- 右侧栏：当前状态、最近更新时间、操作按钮（保存草稿/提交）
- 交互重点：保存反馈要明显（toast或状态提示条）

## 6.3 详情/审核页
- 顶部：稿件标题、状态标签、更新时间
- 主区：正文内容
- 右侧：审核操作区（通过/退回 + 退回意见输入）
- 底部：审核流水时间线（动作、前后状态、意见、时间）

---

## 7. 交互与可用性原则

- 所有关键动作需二次确认或明确提示（尤其退回）。
- 退回意见为空时，前端即时校验并阻止提交。
- 状态驱动按钮显隐：禁止操作不显示或禁用，并给出原因。
- 保证键盘可达性：输入框、按钮支持 `Tab` 导航。

---

## 8. 前端落地建议（原生HTML/JS）

- 使用CSS变量统一管理色板（定义在`:root`）。
- 页面使用通用布局类：`.page`, `.card`, `.toolbar`, `.status-badge`, `.btn`。
- 将状态码与颜色映射维护在一个JS对象中，避免散落硬编码。
